Abstract

We present MCP4IFC, an open-source framework designed to bridge the gap between LLM research and the Architecture, Engineering, and Construction (AEC) industry. It enables AI agents to understand and execute natural language commands within BIM workflows by interacting directly with IFC data models.

The system operates as a Blender-based server that exposes a suite of BIM-specific tools via the Model Context Protocol (MCP). It uses the robust IfcOpenShell library for all read/write operations on IFC files, ensuring data integrity and standards compliance. With MCP4IFC, an LLM can perform a range of actions, from building a semantically correct house from scratch to querying and editing complex, existing IFC models. Key features include a lightweight retrieval system for IFC documentation and a code-generation fallback for creating custom geometry.

As a fully open-source project, MCP4IFC provides a foundation for researchers and developers to create and test novel AI-assisted modeling workflows. Any MCP-compatible AI client can use its tools out-of-the-box, fostering an open ecosystem for innovation in AEC. We also present preliminary results from our extensive benchmarking results evaluating current LLM models for BIM tasks.

Contributions

  • IFC-aware generation: outputs that adhere to IFC entities and relations (draft).
  • Topology- and semantics-consistent reconstruction objectives.
  • Validation protocol aligning geometric fidelity with IFC structural checks.
  • OpenBIM-friendly design compatible with common IFC toolchains.
